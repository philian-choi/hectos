# Hectos Use Case Map Overview

각 퍼소나가 Hectos를 필요로 하는 **구체적인 상황(Use Case)**을 6대 요소로 정리했습니다.

## 📊 UCM 요약 표

| ID | Persona | Core Problem | Why Hectos? | Primary Frequency |
|----|---------|--------------|-------------|-------------------|
| **01** | **Kevin** | 숫자 세기 인지 부하 | 터치 없는 자동 카운팅 + 햅틱 | 매일 1회+ (홈트 루틴) |
| **02** | **Sarah** | 카메라 프라이버시 불신 | 온디바이스 AI, 서버 전송 없음 | 앱 설치/탐색 시점 |
| **03** | **Travis** | 정체기에서 포기 | 6주 완성 + 배치 테스트 | 새해/여름 전 (계절성) |
| **04** | **Elena** | 투박한 UI로 동기 저하 | 프리미엄 Glassmorphism UI | 항상 (앱 열 때마다) |

## 🎯 광고 타겟팅 시사점

### Kevin & Travis (Primary)
- **메시지 핵심**: "숫자는 앱이, 집중은 당신이" / "6주만에 0개→100개"
- **광고 채널**: Instagram Reels, YouTube Shorts (운동 관련 콘텐츠 시청자)
- **타이밍**: 연중 상시 + 1월/5월 집중

### Sarah (Secondary)
- **메시지 핵심**: "Your data stays on your phone. Period."
- **광고 채널**: Reddit r/privacy, 테크 블로그, 앱스토어 설명
- **타이밍**: 데이터 유출 뉴스 직후 리타겟팅

### Elena (Secondary)
- **메시지 핵심**: "운동하고 싶게 만드는 디자인"
- **광고 채널**: Instagram (감성/라이프스타일 피드), Design 커뮤니티
- **타이밍**: 앱스토어 피처링 시즌
